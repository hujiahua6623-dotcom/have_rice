# 部署说明

本文说明如何将「大家来投票」后端与前端部署到生产或测试环境。架构为：**FastAPI（含 WebSocket）+ 数据库** 与 **Vue 3 静态前端**，由反向代理统一域名与 TLS。

## 1. 前置条件

- Python 3.11+（与本地开发一致即可）
- Node.js 20+（仅构建前端时需要）
- 数据库：SQLite（单机小流量）或 **MySQL**（推荐生产，连接串须为 `mysql+pymysql://...`）
- 可选：Nginx、Caddy 等反向代理

## 2. 后端部署

### 2.1 安装与配置

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

编辑 `.env`，**生产环境务必修改**：

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | 数据库连接；MySQL 使用同步驱动 **`mysql+pymysql://`** |
| `JWT_SECRET` | 用户 JWT 签名密钥，使用足够长的随机串 |
| `CORS_ORIGINS` | 前端页面完整源，逗号分隔，如 `https://vote.example.com` |
| `ADMIN_CONSOLE_USERNAME` / `ADMIN_CONSOLE_PASSWORD` | 管理控制台登录，勿使用默认值 |
| `ASSIST_GRACE_SECONDS` | 可选，助力提交宽限秒数 |

### 2.2 数据库迁移

```bash
mkdir -p data   # SQLite 时需要
alembic upgrade head
```

生产环境请以 Alembic 为准，勿仅依赖应用启动时的 `create_all`。

### 2.3 运行方式

**开发/小规模**（单进程，带热重载仅用于调试）：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

**生产推荐**：多 worker 需注意 **WebSocket 与内存房间状态** 当前为**单进程内**维护，多实例时同一房间用户须落到同一进程（粘性会话），否则行为异常。简单做法：

- 单机单 worker 跑 uvicorn；或
- 前置一层 sticky session；或
- 后续改造为 Redis 等共享状态（超出本文范围）。

示例（单 worker）：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 1
```

或使用 `gunicorn` + `uvicorn.workers.UvicornWorker`（同样建议先保持 **workers=1** 除非已解决会话粘性）。

### 2.4 健康检查

- `GET /health` → `{"status":"ok"}`

### 2.5 静态文件与数据目录

- SQLite 时确保 `backend/data/` 可写且持久化挂载。
- 日志与监控按运维规范自行接入。

## 3. 前端部署

### 3.1 构建

```bash
cd frontend
npm ci          # 或 npm install
npm run build
```

产物在 `frontend/dist/`，为静态资源。

### 3.2 与后端的 URL 关系

开发时 Vite 将 `/api`、`/ws` 代理到本机 8000 端口。生产环境通常有两种做法：

**方案 A：同域反向代理（推荐）**

- 浏览器只访问 `https://your-domain`，Nginx 将 `/api`、`/ws` 转到后端，其余返回 `dist` 中的 `index.html`（SPA）。
- 前端已使用 `baseURL: '/api'` 与 `location.host` 构造 WebSocket，**无需改代码**。

**方案 B：前后端不同子域**

- 需改前端 axios `baseURL` 与 WebSocket 域名为后端地址，并在后端 `CORS_ORIGINS`、Cookie（若未来使用）中放行前端源。当前项目以 Bearer Token 为主，主要配置 **CORS** 即可。

### 3.3 Nginx 配置示例（方案 A）

```nginx
server {
    listen 443 ssl;
    server_name vote.example.com;

    root /var/www/hava_rice/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }
}
```

SSL 证书请使用 Let’s Encrypt 或运维既有证书。

## 4. 局域网调试

若需局域网 IP 访问前端，除防火墙外请：

- Vite 开发：`frontend/vite.config.ts` 中已可配置 `server.host: true`；
- 后端：`CORS_ORIGINS` 增加 `http://<本机局域网IP>:5173`（或你的前端源）。

详见仓库根目录 README 与 [接口文档](./接口文档.md) 中的说明。

## 5. 安全检查清单

- [ ] `JWT_SECRET` 已更换为强随机值  
- [ ] 管理控制台账号密码已更换  
- [ ] `CORS_ORIGINS` 仅包含可信前端源  
- [ ] 生产环境使用 HTTPS  
- [ ] 数据库凭据与 `.env` 不入库、不泄露
