# 前后端接口文档

本文描述 HTTP API 与房间 WebSocket 协议，便于联调与二次开发。**字段以服务端 Pydantic 模型与实现为准**；完整请求/响应模型见运行中的 OpenAPI。

- **OpenAPI（推荐）**：启动后端后访问 **`/docs`**（Swagger）或 **`/redoc`**。  
- **API 前缀**：REST 接口均为 **`/api`**。  
- **认证**  
  - **玩家**：`Authorization: Bearer <access_token>`（注册/登录返回的 JWT）。  
  - **管理端**：  
    - 控制台：`Authorization: Bearer <admin_console_token>`（`POST /api/admin/login`）；  
    - 房间操作：`X-Admin-Token: <房间创建时返回的 admin_token>` **或** 携带有效控制台 Bearer（见下表）。

---

## 1. REST 接口一览

### 1.1 通用

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查，返回 `{"status":"ok"}`（无 `/api` 前缀） |

### 1.2 认证 `/api/auth`

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/auth/register` | 无 | 注册，body：`email`, `nickname`, `password` |
| POST | `/api/auth/login` | 无 | 登录，body：`email`, `password` |
| GET | `/api/auth/me` | Bearer 用户 | 当前用户信息 |

**响应**（注册/登录）：`{ "access_token": "...", "token_type": "bearer" }`。

### 1.3 房间（玩家）`/api/rooms`

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/rooms` | 无 | 开放房间列表 |
| POST | `/api/rooms/{room_id}/join` | Bearer 用户 | 加入房间（会退出此前所在房间） |
| POST | `/api/rooms/{room_id}/leave` | Bearer 用户 | 离开房间 |

### 1.4 管理端 `/api/admin`

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/admin/login` | 无 | 控制台登录，`username` / `password` |
| POST | `/api/admin/rooms` | 控制台 Bearer | 创建房间，返回 `room` + **明文** `admin_token`（仅一次） |
| GET | `/api/admin/rooms` | 控制台 Bearer | 所有房间列表 |
| GET | `/api/admin/rooms/{room_id}` | 控制台 Bearer 或 `X-Admin-Token` | 房间信息 |
| PATCH | `/api/admin/rooms/{room_id}` | 同上 | 更新名称、人数上限、`status`（开放/关闭等） |
| DELETE | `/api/admin/rooms/{room_id}` | 同上 | 删除房间 |
| POST | `/api/admin/rooms/{room_id}/poll-types` | 同上 | 新增题目（含选项）；业务上每房仅一题 |
| GET | `/api/admin/rooms/{room_id}/poll-types` | 同上 | 题目列表（含选项） |
| GET | `/api/admin/rooms/{room_id}/poll-types/{type_id}` | 同上 | 单题详情 |
| PATCH | `/api/admin/rooms/{room_id}/poll-types/{type_id}` | 同上 | 更新题目 |
| DELETE | `/api/admin/rooms/{room_id}/poll-types/{type_id}` | 同上 | 删除题目 |
| POST | `/api/admin/rooms/{room_id}/poll-types/{type_id}/options` | 同上 | 新增选项 |
| PATCH | `/api/admin/rooms/{room_id}/poll-types/{type_id}/options/{option_id}` | 同上 | 更新选项 |
| DELETE | `/api/admin/rooms/{room_id}/poll-types/{type_id}/options/{option_id}` | 同上 | 删除选项（至少保留 2 个） |

枚举与字段长度以 `app/schemas/`、`app/models.py` 为准（如 `RoomStatus`、`DisplayStyle`）。

## 2. WebSocket：房间内实时通信

### 2.1 连接 URL

```
ws://<host>/ws/rooms/{room_id}?token=<URL编码后的用户JWT>
wss://<host>/ws/rooms/{room_id}?token=...
```

- 须为**已开放**房间的成员（先 `POST /api/rooms/{id}/join`）。  
- 前端实现：`${proto}//${location.host}/ws/rooms/${roomId}?token=...`（与页面同域时由反向代理转发到后端）。

### 2.2 消息封装（JSON）

**服务端 → 客户端** 与 **客户端 → 服务端** 均使用同一外层结构：

```json
{
  "v": 1,
  "msg_type": "字符串",
  "payload": { },
  "server_ts_ms": 1730000000000
}
```

`server_ts_ms` 由服务端填充；客户端上行可省略或随意，服务端不依赖。

### 2.3 客户端 → 服务端 `msg_type`

| msg_type | payload 要点 | 说明 |
|----------|----------------|------|
| `heartbeat` | 无 | 保活，服务端不回复 |
| `chat_message` | `body` 或 `text`：消息正文 | 房间内聊天，广播 `chat_message` |
| `select_option` | `option_id` | 选择题选项 |
| `confirm` | — | 确认选择 |
| `cancel` | — | 取消（倒计时内或选择阶段，行为见实现） |
| `assist_click` | — | 助力阶段客户端本地累计（具体以后端阶段为准） |
| `assist_commit` | `count`：提交助力次数 | 仅在 `assist_submit_window` 后短窗口内有效 |

### 2.4 服务端 → 客户端 `msg_type`（常见）

| msg_type | 说明 |
|----------|------|
| `room_snapshot` | 进房后首包：房间、题目、阶段、`members`、`peer_states`、`chat_messages`（最近 100 条）、`last_round_summary` 等 |
| `welcome` | 有用户加入广播 |
| `user_left` | 用户离开 |
| `peer_state` | 他人选择/确认状态更新 |
| `select_ack` | 仅发给操作者，确认选项 |
| `phase_changed` | 游戏阶段变化 |
| `countdown` | `phase` + `seconds` 倒计时 tick |
| `assist_submit_window` | `grace_ms` / `grace_sec`，此后可发 `assist_commit` |
| `marquee_start` | 跑马灯：`steps`、`ordered_option_ids`、`winning_option_id`、`duration_sec` |
| `round_result` | 本局结果 |
| `chat_message` | 新聊天消息 |
| `room_closed` | 房间关闭或删除：`reason` |
| `error` | `message`；聊天错误可带 `scope: "chat"` |

阶段字符串与游戏逻辑详见 [项目设计文档](./项目设计文档.md) 与 `app/services/room_runtime.py`。

## 3. 前端项目中的调用位置（参考）

- 玩家 HTTP：`frontend/src/api/http.ts`（`baseURL: '/api'`）  
- 管理端 HTTP：`frontend/src/api/adminHttp.ts`  
- 房间 WS：`frontend/src/views/RoomView.vue`（`connectWs`）

生产环境同域代理时保持 **`/api`** 与 **`/ws`** 与页面同源即可。
